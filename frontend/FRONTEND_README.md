## Overview

QuizGen is a full-featured adaptive learning platform. The frontend is a React 18 SPA delivering:

| Capability | Details |
|---|---|
| **AI Quiz Generation** | Online MCQ / descriptive exams generated by LLM APIs per country & exam type |
| **Offline Quiz Workflow** | Generate → Print PDF → Scan handwritten sheet → AI Vision grades it |
| **Adaptive Quiz** | Real-time difficulty adjustment based on running accuracy |
| **AI Tutor** | Multi-turn chat tutor + deep concept explainer (memory tips, practice questions) |
| **Performance Hub** | Score trend chart, topic heatmap, skills radar, pass-probability risk analysis |
| **AI Recommendations** | Personalised weekly study plans, resources, and meta-learning strategies |
| **Institute Portal** | 3-step exam wizard with AI question generation, class analytics, student leaderboard |
| **Student Exam Portal** | Search exams by ID, attempt without login, receive AI-generated report |
| **Dark / Light Theme** | CSS-variable-driven, persisted to `localStorage` |

## Getting Started

### Prerequisites

- **Node.js** 18 or later
- **npm** 9+ (or yarn / pnpm)
- **QuizGen backend** running at `http://localhost:5000`

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-org/quizgen-frontend.git
cd quizgen-frontend

# 2. Install dependencies
npm install
```

### Environment Variables

Create a `.env` file in the project root:

```env
# Leave empty during local development to use the Vite dev proxy
VITE_API_URL=

# Set to your deployed backend URL for staging / production
# VITE_API_URL=https://api.yourdomain.com
```

When `VITE_API_URL` is empty, Vite proxies all backend routes (`/auth`, `/quiz`, `/institute`, `/student`, `/practice`, `/tutor`, `/offline`, `/health`) to `http://localhost:5000`, eliminating CORS issues without any extra configuration.

### Development Server

```bash
# Start HMR dev server on http://localhost:3000
npm run dev
```

### Production Build

```bash
# Type-check + bundle into dist/
npm run build

# Preview the production bundle locally
npm run preview
```

The `dist/` folder is a fully static SPA. Deploy to any static host (Vercel, Netlify, AWS S3 + CloudFront, etc.). Configure the host to serve `index.html` for all routes (SPA fallback / try_files).

### Dark / Light Theming

The app defaults to **dark** mode. Toggling adds/removes the class `light` on `<html>`, which overrides every CSS custom property defined in `:root`:

```css
/* Default (dark) */
:root {
  --bg:        #0a0f1e;
  --surface:   rgba(255,255,255,0.05);
  --border:    rgba(255,255,255,0.10);
  --text:      #ffffff;
  --text-muted: rgba(255,255,255,0.50);
}

/* Light override */
html.light {
  --bg:        #f1f5f9;
  --surface:   rgba(255,255,255,0.85);
  --border:    rgba(0,0,0,0.08);
  --text:      #0f172a;
  --text-muted: #475569;
}
```

Components consume these via `style={{ color: 'var(--text-muted)' }}`. Tailwind's `darkMode: 'class'` is configured but the primary theming path is the CSS variable system, which supports smooth `transition` on `background-color` and `color`.

Theme preference is persisted to `localStorage` as `qg-theme`.

## Architecture

### State Management

The app uses **React Context + local component state** — no external state library is needed at this scale.

| Context | Scope | Key values |
|---|---|---|
| `AuthContext` | Global | `user`, `isAuth`, `loading`, `login()`, `register()`, `logout()`, `refreshUser()` |
| `ThemeContext` | Global | `theme`, `isDark`, `toggleTheme()` |

Page-level data (exam questions, performance stats, etc.) lives in local `useState` within the relevant page component and is fetched on mount via `useEffect`.

### API Layer

`src/services/api.js` exports a single configured **Axios instance** plus named API modules:

```
authAPI     → /auth/*
quizAPI     → /quiz/*
instituteAPI → /institute/*
studentAPI  → /student/*
practiceAPI → /practice/*
tutorAPI    → /tutor/*
offlineAPI  → /offline/*
```

**Request interceptor** — automatically attaches `Authorization: Bearer <token>` from `localStorage`.

**Response interceptor** — on `401` (outside `/auth/*` routes) clears the token and redirects to `/login`, preventing silent stale-session bugs.

**Error helper:**

```jsx
import { getErrMsg } from '@/services/api';

catch (err) {
  toast.error(getErrMsg(err));
  // Priority: err.response.data.error → .message → err.message → 'Something went wrong'
}
```

### Routing & Guards

Three guard components defined in `App.jsx`:

| Guard | Behaviour |
|---|---|
| `<Guard>` | Requires authenticated user — redirects to `/login` |
| `<GuestOnly>` | Requires unauthenticated state — redirects to `/dashboard` |
| `<InstituteGuard>` | Requires `user.role === 'institute'` — redirects non-institute users to `/dashboard` |

All guards show a full-page `<Spinner>` while `AuthContext.loading` is `true` (JWT verification in flight), preventing flash-of-wrong-route.

### Vite Proxy (Dev)

`vite.config.js` proxies all backend routes to `http://localhost:5000`:

```js
proxy: {
  '^/(auth|quiz|institute|student|practice|tutor|offline|health)': {
    target: 'http://localhost:5000',
    changeOrigin: true,
  },
}
```

This means you never need `CORS` headers or a local `.env` URL during development.

## Context Providers

### `AuthContext`

```jsx
const { user, isAuth, loading, login, register, logout, refreshUser } = useAuth();
```

| Value | Type | Description |
|---|---|---|
| `user` | `object \| null` | Full user document from `GET /auth/me` |
| `isAuth` | `boolean` | `true` when `user !== null` |
| `loading` | `boolean` | `true` while verifying the saved JWT on first mount |
| `login(username, password)` | `async fn` | Throws on failure — caller handles toast |
| `register(formData)` | `async fn` | Throws on failure |
| `logout()` | `async fn` | Calls `POST /auth/logout`, clears `qg-token` + `qg-user` |
| `refreshUser()` | `async fn` | Re-fetches `/auth/me` and updates state + localStorage |

Token lifecycle: stored as `qg-token`, read on every request by the Axios interceptor. User cached as `qg-user` to avoid a blank flash on reload.

### `ThemeContext`

```jsx
const { theme, isDark, toggleTheme } = useTheme();
```

| Value | Type | Description |
|---|---|---|
| `theme` | `'dark' \| 'light'` | Current theme string |
| `isDark` | `boolean` | `true` when theme is dark |
| `toggleTheme()` | `fn` | Swaps theme and persists to `qg-theme` |

## API Services

All modules export plain async functions that return Axios response objects. Call sites destructure `.data`.

```js
import { quizAPI, getErrMsg } from '@/services/api';

const res = await quizAPI.getCountries();
// res.data → array of country objects

try {
```

## Security & Exam Integrity

### `useExamLockdown` Hook

Applied in both `OnlineTest.jsx` and `ExamAttempt.jsx` while the exam is active:

| Lockdown Measure | How |
|---|---|
| Back button blocked | `history.pushState` dummy entry + `popstate` listener |
| Tab switch detected | `document.visibilitychange` event |
| Window blur detected | `window.blur` event |
| Keyboard shortcuts blocked | `keydown` — F12, Ctrl/Meta + T/N/W/Tab, Alt+Tab |
| Context menu disabled | `contextmenu` preventDefault |
| Tab close warning | `beforeunload` returnValue |

Each violation increments a counter displayed in the top bar and shows a red banner for 4 seconds. Violation data is included in the submission payload for backend audit.

### One-Attempt Enforcement

`studentAPI.startAttempt()` returns an error if the student ID has already attempted the exam. `ExamAttempt.jsx` surfaces this with a clear message on the registration form.

## Key Patterns & Conventions

**Path alias `@/`** — configured in `vite.config.js`, resolves to `src/`:
```jsx
import Layout from '@/components/common/Layout';
import { useAuth } from '@/context/AuthContext';
import { quizAPI } from '@/services/api';
```

**CSS variables for theming** — Tailwind cannot resolve runtime CSS vars, so dynamic colours use inline styles:
```jsx
style={{ color: 'var(--text-muted)' }}
style={{ backgroundColor: 'var(--surface)', borderColor: 'var(--border)' }}
```

**Toast error pattern** — every API call wraps errors consistently:
```jsx
catch (err) { toast.error(getErrMsg(err)); }
```

**No HTML `<form>` elements** — all submission logic uses `onClick` handlers to avoid default browser form behaviour in this SPA.

**Shimmer loading** — `StatCard` and list skeletons use `.shimmer` instead of a spinner for inline loading states, matching the card layout before data arrives.

**Role-aware components** — `Dashboard.jsx` renders `<StudentDashboard>` or `<InstituteDashboard>` based on `user.role`. `Navbar.jsx` renders `STUDENT_NAV` or `INSTITUTE_NAV` from the same field.

**Authenticated blob downloads** — PDF downloads in `OfflineQuiz.jsx` go through the Axios instance (not bare `<a href>`), so the `Authorization` header is included and the file downloads with a generated filename.

**Batch API calls** — `Dashboard.jsx` and `PerformanceHub.jsx` use `Promise.all()` to fire multiple independent requests in parallel, reducing total load time.

## Deployment

### Static Hosting (Vercel / Netlify)

```bash
npm run build
# Upload dist/ or connect repo; set build command to "npm run build", output to "dist"
```

Set the rewrite / redirect rule to serve `index.html` for all paths:
- **Netlify:** add `_redirects` file: `/* /index.html 200`
- **Vercel:** add `vercel.json` with `"rewrites": [{"source":"/(.*)", "destination":"/index.html"}]`

### Docker

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

`nginx.conf` must include `try_files $uri /index.html;` for SPA routing.

### Environment Variables at Runtime

For Docker deployments where the API URL is injected at runtime rather than build time, replace `VITE_API_URL` with a runtime config object served from `public/config.js` and loaded in `index.html`.
